CATATAN UPDATE SSO MENTORIZE

Perubahan utama:
1. Login Mahasiswa/Alumni sekarang diarahkan ke SSO UNISA.
2. Login manual mahasiswa/alumni lama disembunyikan dari UI. Fallback darurat masih bisa dibuka melalui /login?manual=1.
3. Admin tetap login manual melalui /admin/login.
4. Route SSO baru:
   - /sso/login
   - /sso/callback
   - /sso/logout
5. /login akan memulai SSO. Jika SSO lama callback ke /login dengan parameter code, sistem meneruskan ke /sso/callback.
6. Logout SSO diarahkan ke logout SSO UNISA, lalu kembali ke /. Ini mencegah logout langsung login ulang otomatis.
7. Tambahan notif_type sesi agar cron tidak gagal: session_started, session_expired, session_extension_requested, session_extension_approved, session_extension_rejected.

System Parameters yang perlu dicek:
- mentorize.sso.client_id = local_mentorize
- mentorize.sso.client_secret = isi client secret SSO
- mentorize.sso.redirect_uri = http://localhost:8069/sso/callback
  Jika server SSO masih mendaftarkan /login, isi: http://localhost:8069/login
- mentorize.sso.post_logout_redirect_uri = http://localhost:8069/
- mentorize.sso.verify_ssl = false untuk development lokal, true untuk production

Mapping role final:
- kdstatus=A => mahasiswa aktif / role mahasiswa
- kdstatus=L => mahasiswa lulus / role alumni
- kdstatus selain A dan L => login ditolak
- loginas/role dari SSO tetap dibaca sebagai konteks, tetapi kdstatus menjadi penentu utama.


UPDATE KDSTATUS FINAL:
- kdstatus=A: masuk sebagai Mahasiswa aktif.
- kdstatus=L: masuk sebagai Alumni / mahasiswa lulus.
- kdstatus selain A dan L: login ditolak dengan pesan status akademik belum didukung.
- Field loginas tetap dibaca sebagai konteks, tetapi role utama ditentukan dari kdstatus.
